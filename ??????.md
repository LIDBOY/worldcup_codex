# 项目基本要求与升级记录

本文件用于记录世界杯预测 AI Agent 的基本要求、每次升级内容、日期和构建思路。以后每次功能更新、架构调整、发布策略变化，都必须追加到本文件，便于回看系统演进。

## 当前基本要求

- 系统必须自动运行，不依赖人工手写比赛结果。
- 系统必须使用真实数据源，不允许 mock data。
- 赛程主源必须来自 FIFA 官方数据。
- 比赛真实性必须至少通过两个来源校验。
- 伤停信息只能来自官方或媒体源；无来源时必须标记 `unknown`。
- 赔率信息只能来自真实赔率源；无赔率时必须标记 unavailable，不允许补假赔率。
- 分析层唯一模型为 `deepseek-v4-pro`。
- 生成层模型为 `deepseek-v4-flash`。
- 必须统计 token usage，并按 `total_tokens * 0.00001` 计算 cost。
- 必须通过 GitHub Pages 展示每日预测页面。
- 必须保留每日运行日志和数据来源信息。

### 2026-06-25 V3：Agent 级系统升级

目标：

- 构建完全自动化的世界杯预测 AI Agent。
- 数据层自动抓 FIFA 官方赛程、伤停信息、赔率数据。
- 至少两个来源校验比赛真实性。
- `deepseek-v4-pro` 作为唯一分析模型。
- `deepseek-v4-flash` 负责网页和 JSON 输出。

构建思路：

- 使用 FIFA official calendar API 作为主赛程来源。
- 使用 ESPN FIFA World Cup scoreboard 作为第二来源交叉校验。
- 从 ESPN odds feed 提取 moneyline，转换为隐含概率和归一化概率。
- 使用 FIFA official search API 和 ESPN World Cup news API 扫描伤停新闻。
- 伤停状态只允许三类：
  - `confirmed injury`：FIFA 官方文章命中。
  - `probable injury`：媒体新闻命中。
  - `unknown`：没有可靠来源。
- 分析输出必须包含胜平负概率、xG、战术对位、伤停修正、爆冷概率、信心等级。
- 新增 09:30 finalize 阶段，专门写入 `data/latest.json` 和 token 统计。

自动流程：

- 08:15：自动抓 FIFA 赛程、伤停、赔率。
- 08:30：`deepseek-v4-pro` 分析。
- 09:00：`deepseek-v4-flash` 生成网页。
- 09:30：写入 JSON 和 token 统计。
- 10:00：部署 GitHub Pages。

输出结构：

```json
{
  "matches": [],
  "analysis_model": "deepseek-v4-pro",
  "render_model": "deepseek-v4-flash",
  "usage": {
    "input_tokens": 0,
    "output_tokens": 0,
    "total_tokens": 0,
    "cost_estimate": 0
  },
  "data_sources": {
    "fifa": true,
    "injury": true,
    "odds": true
  }
}
```

验收重点：

- 自动抓取 FIFA 比赛。
- 自动识别伤停。
- 自动分析赔率。
- 页面展示预测、概率、比分、风险、伤停、赔率、token、cost。
- 每天自动运行并发布到 GitHub Pages。

### 2026-06-28 V3.3：深度研究式分析结构与比分结果 Top 3

修改原因：

- 当前 V3 Agent 已具备 FIFA + ESPN + DeepSeek 双模型生产流程，但每场比赛的分析结构仍偏概览，缺少类似 ChatGPT 深度研究报告的分层分析框架。
- 用户希望保留现有黑底夜间 UI 和 V3 后端架构，同时增强每场比赛的可解释性。
- 比分展示需要从单一主比分升级为“比分结果 Top 3”，但明确禁止输出或展示比分概率。

主要内容：

- 主版本号继续保持 V3，本次为 V3 小版本更新。
- DeepSeek V4-Pro 分析 prompt 升级：
  - 继续要求 `deepseek-v4-pro` 是唯一分析模型。
  - 必须分析所有通过 V3 校验的比赛。
  - 每场比赛新增 `score_options`，包含 3 个比分结果。
  - 第 1 个比分为主推荐比分，另外 2 个为高可能比分。
  - 每个比分只包含 `score`、`rank`、`reason`。
  - 禁止输出、计算或展示比分概率。
- 每场比赛新增 `method_factors` 分析因子面板：

  - `weather_venue`：天气/场地影响，无可靠来源时写 unknown。
  - `tactical_key`：关键战术对位。

  - 主推荐比分视觉更突出。
  - 每个比分下方显示一句原因。
  - 新增可折叠的“分析因子”网格区域。
  - 保留移动端优先、hover 半动画、深蓝卡片、彩色概率条。

  - 校验每个 score option 必须包含 `score`、`rank`、`reason`。
  - 禁止 score option 出现 probability / 概率字段。
  - 校验页面不得包含“比分概率”或 score probability 文案。
  - 校验每场比赛必须有完整 8 项 `method_factors`。

### 2026-06-29 V3.4：对战结构图与可视化解释升级

修改原因：

- V3 Agent 已具备比赛预测、比分 Top 3 和分析因子面板，但用户需要更直观的对战结构表达。
- 本次更新增强每场比赛的可解释性和可视化表达，让双方强弱、攻防路径、关键对位、风险触发点和预测倾向能被快速阅读。
- 主版本号继续保持 V3，不升级到 V4，不改变现有 FIFA + ESPN + DeepSeek 双模型架构。

主要内容：

- 每场比赛新增 `matchup_graph` 数据结构：
  - `nodes`：双方进攻、中场/转换、防守等结构节点。
  - `edges`：至少 3 条关键对战关系。
  - `summary`：结构图总结。
  - `advantage_side`：优势方，取值为 `home`、`away` 或 `balanced`。
  - `key_battle`：关键对位。
  - `risk_trigger`：风险触发点。
- DeepSeek V4-Pro 分析 prompt 升级：
  - 继续要求 `deepseek-v4-pro` 是唯一分析模型。
  - 必须分析所有通过 V3 校验的比赛。
  - 保留胜平负概率、xG、爆冷概率、信心等级、风险分析、赔率对比、伤停修正、`score_options`、`method_factors`。
  - 每场比赛必须输出 `matchup_graph`。
  - 结构图中的 `weight` 只能表示 0-100 相对分析强度，不代表概率、胜率、对位概率或比分概率。
  - 禁止输出“结构图概率”“对位概率”“比分概率”以及 matchup probability / score probability 文案。
- 页面 UI 在黑底夜间风格中新增“对战结构图”模块：
  - 使用 `<details>` 折叠区域，避免页面过满。
  - 显示优势方、关键对位、风险触发点和结构图总结。
  - 显示主队/客队节点区：进攻、中场/转换、防守。
  - 显示关键关系卡，并按 impact 高/中/低使用不同颜色强调。
  - 继续保持移动端优先、hover 半动画、长文本自动换行。
- `data/latest.json` 继续保持 V3 contract，并为每场比赛追加：
  - `matchup_graph`
- `app/validate.py` 校验规则升级：
  - 校验每场比赛必须有 `matchup_graph`。
  - 校验 `nodes` 至少 6 个节点，`edges` 至少 3 条关系。
  - 校验 node 必须包含 `id`、`label`、`type`、`side`、`weight`、`note`。
  - 校验 edge 必须包含 `from`、`to`、`label`、`impact`、`direction`、`note`。
  - 校验 `impact` 只能是 `high`、`medium`、`low`。
  - 校验页面必须包含“对战结构图”“关键对位”“风险触发点”。
  - 校验页面不得包含“结构图概率”“对位概率”“比分概率”或 matchup probability / score probability。

V3 Agent 后端保持不变：

- FIFA 官方赛程仍是主源。
- ESPN 仍用于第二来源校验和赔率。
- 伤停、赔率、天气、排名不得编造；无可靠来源时按 unknown / unavailable 展示。
- 分析模型仍为 `deepseek-v4-pro`。
