# 项目基本要求与升级记录

本文件用于记录世界杯预测 AI Agent 的基本要求、每次升级内容、日期和构建思路。以后每次功能更新、架构调整、发布策略变化，都必须追加到本文件，便于回看系统演进。

## 当前基本要求

- 系统必须自动运行，不依赖人工手写比赛结果。
- 系统必须使用真实数据源，不允许 mock data。
- 赛程主源必须来自 FIFA 官方数据。
- 比赛真实性必须至少通过两个来源校验。
- 伤停信息只能来自官方或媒体源；无来源时必须标记 `unknown`。
- 赔率信息只能来自真实赔率源；无赔率时必须标记 unavailable，不允许补假赔率。
- 分析层唯一模型为 `deepseek-v4-pro`。
- 生成层模型为 `deepseek-v4-flash`。
- 必须统计 token usage，并按 `total_tokens * 0.00001` 计算 cost。
- 必须通过 GitHub Pages 展示每日预测页面。
- 必须保留每日运行日志和数据来源信息。

## 版本记录

### 2026-06-24 V1：自动化 GitHub Pages 世界杯预测系统

目标：

- 建立 GitHub Actions 自动运行流程。

构建思路：

输出：

### 2026-06-25 V2：DeepSeek V4 双模型生产级升级

目标：

- 分析层使用 `deepseek-v4-pro`。
- 生成层使用 `deepseek-v4-flash`。
- 页面内容和国家队名称改为中文。
- 增加 token usage 和 cost 展示。
- 使用 GitHub Pages 自动发布。

构建思路：

- 将流程拆成 research、analysis、render、publish。
- analysis 阶段调用 `deepseek-v4-pro`，开启 high reasoning 和 thinking。
- render 阶段调用 `deepseek-v4-flash` 生成 HTML。
- 从 DeepSeek `response.usage` 提取 input/output/total tokens。
- `data/latest.json` 写入 analysis、render、usage、model 等字段。

输出：

- 中文预测页面。
- 双模型调用链。
- Token 和 cost 面板。
- GitHub Actions 定时运行。

### 2026-06-25 V3：Agent 级系统升级

目标：

- 构建完全自动化的世界杯预测 AI Agent。
- 数据层自动抓 FIFA 官方赛程、伤停信息、赔率数据。
- 至少两个来源校验比赛真实性。
- `deepseek-v4-pro` 作为唯一分析模型。
- `deepseek-v4-flash` 负责网页和 JSON 输出。

构建思路：

- 使用 FIFA official calendar API 作为主赛程来源。
- 使用 ESPN FIFA World Cup scoreboard 作为第二来源交叉校验。
- 从 ESPN odds feed 提取 moneyline，转换为隐含概率和归一化概率。
- 使用 FIFA official search API 和 ESPN World Cup news API 扫描伤停新闻。
- 伤停状态只允许三类：
  - `confirmed injury`：FIFA 官方文章命中。
  - `probable injury`：媒体新闻命中。
  - `unknown`：没有可靠来源。
- 分析输出必须包含胜平负概率、xG、战术对位、伤停修正、爆冷概率、信心等级。
- 新增 09:30 finalize 阶段，专门写入 `data/latest.json` 和 token 统计。

自动流程：

- 08:15：自动抓 FIFA 赛程、伤停、赔率。
- 08:30：`deepseek-v4-pro` 分析。
- 09:00：`deepseek-v4-flash` 生成网页。
- 09:30：写入 JSON 和 token 统计。
- 10:00：部署 GitHub Pages。

输出结构：

```json
{
  "matches": [],
  "analysis_model": "deepseek-v4-pro",
  "render_model": "deepseek-v4-flash",
  "usage": {
    "input_tokens": 0,
    "output_tokens": 0,
    "total_tokens": 0,
    "cost_estimate": 0
  },
  "data_sources": {
    "fifa": true,
    "injury": true,
    "odds": true
  }
}
```

验收重点：

- 自动抓取 FIFA 比赛。
- 自动识别伤停。
- 自动分析赔率。
- 页面展示预测、概率、比分、风险、伤停、赔率、token、cost。
- 每天自动运行并发布到 GitHub Pages。

### 2026-06-25 V3.1：北京时间比赛日与旧版 UI 风格恢复

修改原因：

- V3 Agent 后端能力已经完成，但页面视觉风格偏离了 V3 之前更简洁的卡片布局。
- 页面直接展示 UTC 时间，不符合中文用户阅读习惯。
- 比赛分块需要按中国观赛语境处理，而不是自然日 00:00-24:00。

主要内容：

- 页面所有可见比赛时间改为北京时间 / Asia/Shanghai。
- 比赛按“中国时间比赛日”分块：
  - 每个比赛日从北京时间当天 18:00 开始。
  - 到第二天 18:00 之前结束。
  - 例如北京时间 6月26日 01:00 的比赛归入 6月25日比赛日。
- 分块内比赛按北京时间升序排序。
- 页面恢复 V3 之前的浅色旧版 UI 风格：
  - 顶部状态区。
  - 统计卡。
  - 8px 圆角比赛卡。
  - 胜平负进度条。
  - 清晰的来源与风险信息。
- V3 Agent 后端保持不变：
  - FIFA 官方赛程仍是主源。
  - ESPN 仍用于第二来源校验和赔率。
  - 伤停信息仍按官方/媒体/unknown 规则展示。
  - 分析模型仍为 `deepseek-v4-pro`。
  - 生成模型仍为 `deepseek-v4-flash`。
  - 不使用 mock data，不手写比赛结果，不跳过 DeepSeek API。

实现说明：

- render 阶段继续调用 `deepseek-v4-flash` 并统计真实 token。
- 最终页面使用固定旧版风格模板渲染，以保证北京时间、比赛日分块和 UI 一致性。
- `data/latest.json` 继续保持 V3 contract，并额外保留北京时间展示字段和比赛日元数据。

### 2026-06-25 V3.2：黑底夜间 UI 与两个中国比赛日展示

修改原因：

- 用户更喜欢上一版预测卡片的清晰信息层级，但希望视觉参考移动端截图，改为黑底、夜间、深蓝卡片和霓虹点缀。
- V3 Agent 后端能力必须保持稳定，不升级主版本，不改成 V4。
- 页面不能因为默认 1 天窗口只展示部分比赛，需要覆盖“当前中国比赛日”和“下一个中国比赛日”的全部双源校验比赛。

主要内容：

- 主版本号保持 V3，本次为 V3 小版本更新。
- 页面改为黑底夜间风格：
  - 深色背景。
  - 深蓝 / 蓝黑预测卡片。
  - 绿色、黄色、红色分别强调主胜、平局、客胜概率。
  - Token / Cost 使用深色统计卡展示。
  - 比赛卡片保留清晰比分、概率条、战术、风险、置信度和比赛 ID。
- 增加轻微 hover 半动画反馈：
  - 卡片悬停轻微上浮。
  - 边框和阴影轻微发光。
  - 概率条亮度增强。
  - Token 卡片悬停有轻微反馈。
- 页面顶部信息面板展示：
  - `2026 世界杯 · 预测分析`。
  - 当前展示的两个中国比赛日范围。
  - 生成时间，显示北京时间。
  - 分析模型 `deepseek-v4-pro`。
  - 生成模型 `deepseek-v4-flash`。
  - 总比赛场次。
- 比赛显示范围改为两个“中国比赛日”：
  - 从预测生成时的北京时间计算当前比赛日。
  - 当前比赛日和下一个比赛日均展示。
  - 每个比赛日按北京时间 18:00 到次日 18:00 划分。
  - 分块内按北京时间升序排序。
  - render 层不截断比赛列表，遍历两个比赛日内所有通过 V3 数据校验的比赛。
- research 阶段不再使用默认 1 天自然日窗口：
  - 先计算两个中国比赛日。
  - 再反推需要抓取的 FIFA UTC 日期跨度。
  - FIFA 官方赛程仍为主源。
  - ESPN 仍用于第二来源校验和赔率。
  - 无真实伤停显示 `unknown`。
  - 无真实赔率显示 unavailable。

V3 Agent 后端保持不变：

- 分析层唯一模型仍为 `deepseek-v4-pro`。
- 生成层仍为 `deepseek-v4-flash`。
- 不使用 mock data。
- 不手写比赛结果。
- 不伪造伤停或赔率。
- 不跳过 DeepSeek API。
- `data/latest.json` 继续满足 V3 contract，并额外保留 `display_window` 与 `china_match_days` 便于验证两个中国比赛日。
