# V3 完赛真实赛果展示设计

## 目标

在不改变 V3 Agent 模型与数据源架构的前提下，让已完赛比赛在比赛卡片和淘汰赛 bracket 节点中优先显示官方真实比分。只有尚未开赛或对阵尚未确定的比赛显示未开赛状态或待定。

## 数据来源与判定

- FIFA 官方赛程仍是主源，使用官方比赛状态和主客队 `Score` 作为赛果依据。
- ESPN 仍为第二来源，读取比赛状态和主客队比分用于辅助校验与补充，不覆盖有效的 FIFA 官方赛果。
- 系统建立统一的 `result` 对象，包含 `status`、`completed`、`home_score`、`away_score`、`display`、`source` 和可选的第二来源校验状态。
- 只有官方状态明确完赛且双方比分均为有效非负整数时，才视为可展示的真实赛果。
- 未开赛比赛不把接口中的默认 `0-0` 当成真实赛果；对阵未确定节点保持 `待定`。

## 页面行为

- 已完赛比赛卡片的主比分区显示 `完赛 · 真实比分` 和真实比分。
- 未完赛比赛卡片的主比分区显示 `未开赛 · 预测比分` 和模型预测比分。
- 已完赛卡片继续保留胜平负概率、前 3 预测比分和分析字段，并明确标注为 `赛前预测`，用于复盘。
- bracket 中已完赛真实节点显示 `完赛` 与真实比分；未完赛真实节点显示赛程状态，只有占位节点显示 `待定`。
- 真实比分和预测比分使用不同标签与颜色，避免语义混淆。

## Contract 与验证

- `matches[*]` 和 `tournament_structure` 真实节点保留统一 `result` 数据。
- 校验器要求 `completed=true` 时必须存在合法主客队真实比分；`completed=false` 时不得把比分标记为真实赛果。
- HTML 校验覆盖 `完赛 · 真实比分`、`未开赛 · 预测比分`、`赛前预测` 和 bracket 完赛结果。
- 增加回归测试覆盖 FIFA 完赛、ESPN 辅助校验、未开赛 0-0、防止占位节点伪造赛果，以及两种页面渲染状态。

## 保持不变

- FIFA 主源、ESPN 第二来源与赔率。
- 分析模型仅使用 `deepseek-v4-pro`，生成模型使用 `deepseek-v4-flash`。
- 不使用 mock 生产数据，不伪造赛果、赔率、伤停、天气或排名，不跳过 DeepSeek API。
